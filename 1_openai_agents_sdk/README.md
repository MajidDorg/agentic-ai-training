# Module 1 — OpenAI Agents SDK

The first framework of the course. You'll learn the OpenAI Agents SDK through three short labs, then build your own project in Lab 4.

## The labs

| Lab | File | What you learn |
|---|---|---|
| **1** | [1_lab1_agents.ipynb](1_lab1_agents.ipynb) | What an agent is; building & running one; **async** and parallel runs |
| **2** | [2_lab2_cold_email.ipynb](2_lab2_cold_email.ipynb) | **Tools**, **agents-as-tools**, **handoffs** (an automated sales SDR) |
| **3** | [3_lab3_structured_guardrails.ipynb](3_lab3_structured_guardrails.ipynb) | **Structured outputs** & **guardrails** |
| **4** | *you build it* | Pick a project, sketch the architecture, let Codex build it, compare |

Open each notebook, select the `.venv` kernel, and run cells with **Shift + Enter**.

## Lab 4 — your project

After Lab 3 you'll have the full toolkit: agents, prompts, async, tools, agents-as-tools, handoffs, structured outputs, guardrails.

1. Your instructor presents **four project options** (by their end result).
2. You **sketch the architecture** on paper — which agents, tools, handoffs, guardrails, structured outputs.
3. You tell **Codex** your choice; it builds the Lab 4 notebook (within what Labs 1–3 taught — see [AGENTS.md](AGENTS.md)).
4. You **compare** your sketch with Codex's build — same? different? why?
5. Then Codex turns it into a **running app with a GUI**.

The point is to think like an **architect** and learn to **read and judge** what the AI produces — not to memorize syntax.

The four options are: **Customer Support Assistant · Social Content Studio · Trip / Event Planner · Smart Inbox Assistant.** Full specs are in [AGENTS.md](AGENTS.md) (that's the file Codex reads).
